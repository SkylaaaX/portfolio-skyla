VR-Store (site e-commerce)

Présentation

VR-Store est une application e-commerce orientée sur la VR, conçue avec une architecture full-stack claire :
- API Back-end (Node.js, Express, Sequelize, MySQL)
-Front-end (React + Vite).
Le projet a été pensé pour un rendu simple et robuste.

Architecture globale

  • Back-end : API REACT (authentification, produits, administration), modèles Sequelize, connexion MySQL.
  • Front-end : pages React (Home, Catalogue, Panier, Connexion, Inscription, MonEspace, Admin).
  • Flux : le front consomme les endpoints API pour afficher les produits, gérer le panier et piloter la session utilisateur.
  • Objectif : séparation claire de la logique métier, de la donnée et de l’interface.

Cahier des charges

  • Mettre en place une inscription / connexion utilisateur sécurisée.
  • Permettre la consultation d’un catalogue produits VR.
  • Gérer un panier persistant (ajout, suppression, total) via localStorage.
  • Proposer un espace utilisateur (session active, déconnexion).
  • Implémenter un espace administration avec CRUD produits.
  • Conserver un code lisible, maintenable et présentable pour l’épreuve BTS.

Captures d'écran

  • Home : présentation du magasin VR-Store.
  • Catalogue : affichage des produits et ajout au panier.
  • Panier : récapitulatif des articles, suppression, total.
  • Connexion / Inscription : authentification et création de compte.
  • MonEspace : informations de session et actions utilisateur.
  • Admin : gestion CRUD des produits via API.

MCD / MLD

La modélisation repose sur des entités centrales comme User, Produit, RefreshToken et LoginAttempts, ainsi que leurs relations métier. Le passage du MCD au MLD a permis une implémentation cohérente sous Sequelize/MySQL, exploitable directement par l’API.
